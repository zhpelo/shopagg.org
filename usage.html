使用文档

从安装到上线的后台操作手册

这份文档面向网站管理员、运营人员和销售跟单人员，覆盖首次部署、后台日常维护、询盘处理、主题区块配置、权限设置和上线前检查。

适用范围
  • 适用于 SHOPAGG B2B Website v1.1.x，当前源码版本为 v1.1.9。
  • 适合管理员、内容运营、产品维护人员和销售跟单人员阅读。
  • 后台入口通常为 /admin/login，首次登录后应立即修改默认密码。
  • 完整源码和原始文档位于 GitHub 仓库的 Documents/ 目录。

目录

安装部署 首次登录 后台导航 产品与内容 收件箱与询盘 外观区块 系统设置与权限 上线检查

安装部署

SHOPAGG 的部署目标是尽量接近传统 PHP 虚拟主机体验：不需要 Composer、Node.js 或前端构建流程，准备 PHP 与 SQLite 扩展后即可运行。

项目 最低要求 说明
PHP 8.1+ 推荐使用更新的 PHP 8.x 版本。
PHP 扩展 sqlite3、mbstring、fileinfo SQLite 数据、字符串处理和上传 MIME 校验依赖这些扩展。
Web 服务器 Apache 或 Nginx Apache 需启用 mod_rewrite；Nginx 使用 try_files 转发到 index.php。
磁盘权限 storage/、uploads/ 可写 数据库、区块配置、上传文件、备份和日志都写入这两个目录。 
# 1. 获取源码
git clone https://github.com/zhpelo/shopagg-b2b-website.git
cd shopagg-b2b-website

# 2. 设置运行目录权限
chmod -R 755 storage uploads

# 3. 访问站点域名
# 系统会自动创建 SQLite 数据库、执行迁移并初始化默认数据
生产环境注意：确认 storage/ 目录不能被 Web 直接访问，并在上线后启用 HTTPS、关闭调试模式、定期备份 storage/site.db 与 uploads/。

首次登录

首次安装后访问后台登录页，默认路径通常为 /admin/login。默认账号以项目 README 为准，常见初始化账号为 admin / admin123。

立即修改密码

不要让默认管理员密码留在生产环境。登录后进入个人资料或员工管理，先替换强密码。

完善站点资料

在系统设置中填写网站名称、Logo、Favicon、公司简介、联系方式和 SEO 信息。

确认当前主题

进入「外观区块 → 网站模版」，确认启用主题与预期一致，再配置模板区块。

测试表单

前台提交联系表单和产品询盘，确认后台收件箱能够正常收到并处理。

后台导航

后台功能按日常运营流程拆分。管理员可以看到全部模块，普通员工只会看到被授权的模块。

菜单 用途
仪表盘 查看产品、文章、案例、询盘、留言和近期互动统计。
收件箱 处理联系留言和产品询单，跟进客户需求。
产品中心 维护产品、产品分类、图片、摘要、标签、供应商和阶梯价格。
内容管理 发布文章、案例、独立页面，并通过媒体库管理图片和视频。
员工管理 创建后台账号，按职责分配模块权限。仅管理员可见。
外观区块 管理网站模版、模板区块、菜单和轮播图。仅管理员可见。
系统设置 维护基础设置、公司资料、贸易能力、联系方式、翻译、自定义代码和程序更新。

产品与内容

产品和内容是前台展示的基础。建议先整理分类结构，再批量补充产品、文章、案例和页面。

产品中心

  • 产品标题、描述、摘要、分类、标签、供应商和 SEO 字段。
  • 产品图片集与详情图用于前台展示。
  • 阶梯价格支持按采购数量展示不同报价。
  • Slug 只允许小写字母、数字和连字符。

内容管理

  • 文章适合公司新闻、行业资讯、采购指南和展会动态。
  • 案例适合客户项目、定制交付和出口能力展示。
  • 页面适合关于我们、服务介绍、FAQ 等固定内容。
  • 媒体库支持文件夹、搜索、筛选和统一选择弹窗。
Slug 建议：使用简短英文，例如 stainless-steel-pipe、industrial-valve。系统会自动转小写并过滤非法字符，稳定的 Slug 更利于 SEO 和外链分享。

收件箱与询盘

收件箱分为联系留言和产品询单。留言通常来自联系表单，询单通常来自产品详情页，包含更完整的采购上下文。

联系留言

查看客户姓名、邮箱、电话、公司和留言内容。无效留言可以直接删除，重要留言建议同步到内部 CRM 或客户表。

产品询单

通常包含需求产品、数量、公司、联系方式、来源页面和 IP 信息，适合销售跟进。

状态 推荐含义
待处理 新询单，还未开始跟进。
已联系 已经首次联系客户，等待进一步沟通。
已报价 已发出正式报价或方案。
已关闭 成交结束、无效需求或明确不再跟进。

需要交给 Excel 或 CRM 继续处理时，可在询单管理中导出 CSV。

外观区块

外观区块是当前版本的重点能力。管理员可以在不改代码的情况下维护主题、页面文案、菜单和轮播图。

网站模版

系统会读取 /themes 目录下的主题，支持查看预览图、上传 zip 主题包、启用主题和删除未使用主题。

模板区块

按首页、关于页、联系我们、全站通用、品牌与样式等分组编辑标题、描述、按钮、图片、图标和颜色。

菜单管理

支持多个菜单、多级菜单、拖拽排序、自定义链接和新窗口打开。常用 Slug 为 main-nav 与 footer。

轮播图

每个轮播图可以维护多张图片、标题、说明、按钮、跳转链接和排序，适合首页首屏或页面 Banner。

主题上传规则：zip 包里只能有一套主题，主题根目录必须包含 style.css，且 style.css 顶部至少包含 Theme Name 元数据。预览图使用主题目录下的 screenshot.jpg。

系统设置与权限

上线前至少检查基础设置、公司简介和联系方式。多人协作时，不要共用管理员账号，应按职责创建 Staff 账号。

设置页 维护内容
基础设置 网站名称、标语、启用主题、SEO、Logo、Favicon、分享图。
公司简介 / 贸易能力 主营产品、公司介绍、员工人数、厂房面积、出口市场、生产和交付能力。
联系方式 电话、邮箱、地址、社交账号、WhatsApp 等客户触达方式。
翻译设置 前台 Google Translate 在线翻译显示方式和语言配置，支持 249 种语言。
自定义代码 自定义 head/footer 代码，例如统计、验证或客服脚本。
程序更新 检查版本、安装更新、查看备份和数据库迁移状态。

权限建议

  • 运营：开放内容管理、产品中心。
  • 销售：开放收件箱和询盘相关模块。
  • 技术或管理员：开放系统设置、外观区块和程序更新。

安全建议

  • 避免多人共用管理员账号。
  • 定期修改密码。
  • 按“够用即可”原则分配 Staff 权限。

上线检查

正式上线前建议逐项检查，避免默认账号、空联系方式或未测试表单直接暴露给客户。

已修改默认管理员密码，并为员工创建独立账号。
网站名称、Logo、Favicon、SEO、联系方式和公司资料已经填写完整。
至少发布了基础页面、核心产品、产品分类、文章或案例内容。
当前主题正确启用，模板区块文案和图片已替换为正式内容。
菜单和轮播图已经配置完成，移动端展示无明显问题。
联系留言和产品询盘流程已经测试，后台能收到并更新状态。
HTTPS 已启用，storage/ 已禁止 Web 直接访问。
已制定 storage/site.db 与 uploads/ 的备份策略。
